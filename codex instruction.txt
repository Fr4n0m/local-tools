# 🧰 LocalTools

LocalTools is a client-side developer toolbox that provides fast, private, and useful utilities that run entirely in the browser.

---

## Product Positioning

LocalTools is:

- A private developer toolbox
- A browser-based utility hub
- A fast, lightweight, no-backend solution

### SEO Title

LocalTools – Developer Toolbox (Private & Browser-Based Tools)

### Core Value

- All tools run locally in your browser
- No uploads
- No tracking
- Full privacy by design

---

## Core Principles

- 100% client-side processing
- No backend
- No database
- No authentication
- No AI integration
- Privacy-first design
- Lightweight and fast
- Modular and scalable architecture

---

- Next.js (App Router)

- Use App Router (/app)

## Application Structure

The app is a single-page toolbox.

### Layout

- Sidebar (left) → tool list
- Top bar → search + controls
- Main area → active tool

### Rules

- No routing per tool
- No page reloads
- Instant switching
- State must feel app-like

---

## URL State (Tool Selection)

The selected tool must be reflected in the URL using query parameters.

Example:

/?tool=json-formatter

### Behavior

- Selecting a tool updates the URL
- Loading the app reads the tool from the URL
- Invalid tool → fallback to default
- No page reloads

### Rules

- Do not create routes per tool
- Use query params only
- Keep UI and URL in sync

---

## Tool Registry System

All tools must be registered in a central registry.

Tool structure:

type Tool = {
  id: string
  name: string
  description: string
  component: React.ComponentType
}

Registry:

export const tools: Tool[] = [...]

### Rules

- Sidebar is generated from registry
- Search uses registry
- Add tool = register once
- No duplication

## File Handling & Privacy

All file processing must happen 100% in the browser.

### Input

- Drag & Drop
- File input

### Processing

- Use browser APIs:
  - File
  - Blob
  - ArrayBuffer
  - Canvas

- Never send files to a server

### Output

- Generate files in memory
- Allow direct download

### Privacy Guarantee

- All data stays in the browser
- No uploads
- No external processing
- No tracking of file content

This must be visible in the UI.

## Modular Translations

Translations must follow module structure.

Example:

src/modules/image-converter/presentation/i18n/en.json  
src/modules/image-converter/presentation/i18n/es.json  

Shared:

src/shared/presentation/i18n/en.json  
src/shared/presentation/i18n/es.json  

### Rules

- Each module owns its translations
- No global monolithic JSON
- Always implement both languages

## Design System

The application must use a consistent and well-defined color system.

### Color Palette

Primary palette:

- Ink Black → #001219
- Dark Teal → #005f73
- Dark Cyan → #0a9396
- Pearl Aqua → #94d2bd
- Wheat → #e9d8a6
- Golden Orange → #ee9b00
- Burnt Caramel → #ca6702
- Rusty Spice → #bb3e03
- Oxidized Iron → #ae2012
- Brown Red → #9b2226

### Design Intent

- Dark tones → base UI and structure
- Teal/cyan tones → primary actions and interactive elements
- Light tones → backgrounds and contrast
- Warm tones (orange/red) → highlights, warnings, and accents

---

## Theming

The application must support both **light mode** and **dark mode**.

### Implementation

- Use Tailwind CSS with CSS variables
- Do not hardcode colors in components
- All colors must be mapped to semantic tokens

---

## Semantic Color Tokens

Do not use raw palette colors directly in components.

Instead, define semantic tokens such as:

- background
- foreground
- primary
- secondary
- accent
- muted
- border
- destructive

---

## Example Token Mapping

### Light Mode

- background → Wheat (#e9d8a6)
- foreground → Ink Black (#001219)
- primary → Dark Cyan (#0a9396)
- secondary → Pearl Aqua (#94d2bd)
- accent → Golden Orange (#ee9b00)
- muted → Pearl Aqua (#94d2bd)

## Styling Rules

- Always use semantic tokens (e.g. bg-background, text-foreground)
- Never use raw hex values in components
- Never use Tailwind default color classes (e.g. text-blue-500)
- Keep color usage consistent across the app
